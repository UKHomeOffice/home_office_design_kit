Badges

Badges provide quick identification of numeric values like tallies or quantities.

Example component

{% include "includes/badges.html" %}

Content patterns

  • badges should only contain numbers
  • badges should always be on the left and at the start of content
  • the badge (a number) should work with any content that follows

Interaction patterns

  • badges shouldn't look or act like buttons
  • badges are for giving insight
  • badges shouldn't be permanent

Inclusive patterns

  • badges shouldn't be interactive and therefore do not need aria labelling
  • the content of the badge repeating
  • colour combinations must meet the minimum WCAG 2.0 AA colour contrast check
  • badges should not use colour only to identify grouping or importance

User research

  • badges should only be used when a user needs to do an action

Services using this pattern

  • Syrian Refugee project
