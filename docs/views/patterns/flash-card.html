Flash card

Flash cards interupt the user flow with important information.

Example component

{% include "includes/flash_card.html" %}

Content principles

  • as this replaces a page follow usual content guidance

Interaction principles

  • should be used to draw the attention of the user to some important information
  • cards should have headings. You may want to visually hide the headings but as a general rule it's simpler not to
  • one thing per card
  • use larger text (24px) in cards
  • use "Continue" buttons to advance to the next card or page
  • don't use form elements (except buttons) within cards. Form elements will add extra complexity and you will need to deal with error states.

User research

  • we have found users can take information in more consistantly when we interupt their flow in this way
