Content principles

  • Pick a good service name. Find out more about naming your service in the Service Manual.
  • For internal systems, only use the 'Digital by default' logo if your service has a public-facing element on GOV.UK
  • For internal systems, there is no need to use th GOV.UK blue bar which is for "branding" only.

Interaction principles

  • The service name should link to the logical starting (home) page.
  • If users need to sign in to the service under different roles, ensure the starting page is appropriate for them.
  • The header can be used for global actions. For example, search and sign out.
  • Do not add more than two global actions, unless their is a proven need to do so.
