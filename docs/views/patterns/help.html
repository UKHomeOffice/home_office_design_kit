Contextual help

Guidance to help users with more complex interactions

Example component

Content principles

  • help content should be easy to comprehend but descriptive.

Interaction principles

  • help should be available to users without interfering in their current flow.
  • help should be easy to close, move and minimise until the user needs it.
  • links to more comprehensive help should be easy to find, if contextual help doesn't answer a users question.

Accessibility principles

User research

  • users find side-by-side guidance more helpful and easier to use. Small overlay windows achieve this while creating the smallest interference with the users flow.
