More about why we use icons

The benefit of icons are categorised as

  • helping users navigate through the service
  • controllers to help users to sort, organise and adjust the information in the service, this can be a filter, closing modals or revealing addition content
  • quick reference is the most subjective area and is relevant to the users, frequency of interaction with the service and reason for needing a quick reference icon/image.

The risk with using icons for quick reference is that users won't understand or misinterpret what the reference is. The options are similar and therefore either the same icons are used or additional icons are needed with only slight changes which are hard to differentiate and increase cognitive load and reduce comprehension.

Identifying whether your icons fit into one of these uses will help to answer whether it's appropriate to use it.

Content patterns

  • any icon should use a visible and audible label to avoid confusion over what it is representing
  • align icon in a central position in relation to any type
  • type can sit to the left or right hand side of the icon

Interaction patterns

  • minimum size for an icon is 16px
  • if it's part of a button the hit area should be 36px
  • the icon shouldn't change colour in a hover/active/focus states, only the background
  • in the normal state, the icon can be a different colour using the GOV.UK colour palette - both in solid and reversed out state - as long as the colour combination meets the minimum WCAG 2.0 AA colour contrast check

Inclusive patterns

  • icons should always have a label and use aria-hidden="true"
  • if an icon can't have a label, ensure it has screenreader only support
    <span class="visuallyhidden">Home button</span>

User research

  • icons have proved supportive with the deaf community, as BSL is a visual method of communication
