Labels

Labeling are used to show classification and marking ownership, they can be useful for users to quickly reference related items.

Example component

{% include "includes/labels.html" %}

Content patterns

  • labels should be single words
  • labels should be common nouns
  • content on labels doesn't have to be descriptive
  • labels are recommended to be lowercase

Interaction patterns

  • labels shouldn't look like buttons
  • labels are for quick referencing
  • labels shouldn't be permanent
  • labels should appear at the end of the content that it labels
  • the margin to the right should be greater to indicate what it relates to

Inclusive patterns

  • labels shouldn't be interactive and therefore do not need aria labelling
  • colour combinations must meet the minimum WCAG 2.0 AA colour contrast check
  • labels should not use colour only to identify grouping or importance

User research

  • adding labels to everything is counterintuative

Services using this pattern

  • HOC
  • Syrian Refugee project
